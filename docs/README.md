## 핵심 기능

고객이 방문 날짜와 메뉴를 선택하면 해당되는 이벤트 혜택들을 미리 안내해주는 기능

## 필요한 기능

### App 클래스

프로그램 실행 로직 담당

### Order 클래스

주문서 인스턴스를 생성하는 클래스로, 주문 목록과 가격들을 필드를 가지고 있다.

- `getOrderList()`: 주문 목록을 문자열로 가공해 반환

#### 가격들을 반환하는 메서드

- `getBeforeDiscountTotalAmount()`: 할인 전 총 주문 금액
- `getTotalDiscountAmount()`: 총 혜택 금액
- `getFinalPaymentAmount()`: 할인 적용 후 최종 결제 금액

#### 가격들을 구해 저장하는 메서드

- `calculateBeforeDiscountTotalAmount()`: 할인 전 총 주문 금액
- `calculateTotalDiscountAmount()`: 총 혜택 금액
- `calculateFinalPaymentAmount()`: 할인 적용 후 최종 결제 금액

### EventPlanner 클래스

이벤트 플래너 인스턴스를 생성하는 클래스로, 크게 다음과 같은 역할을 한다.

1. 방문 날짜와 주문 목록을 가지고 해당되는 이벤트들을 적용
2. 혜택받은 정보들을 반환하는 메서드 보유

- `applyAllEvents()`: 적용 가능한 이벤트를 찾아 주문서를 업데이트
- `setEventsByDate()`: 방문 날짜에 맞는 이벤트를 적용
- `getDayType()`: 방문 날짜가 평일인지 주말인지 판별
- `discountDday()`: 크리스마스 디데이 할인
- `discountWeekly()`: 평일/주말 할인
- `discountSpecialDate()`: 특별 할인
- `setGiftEvent()`: 할인 전 총 주문 금액을 받아 증정 이벤트 적용
- `setEventBadge()`: 총 혜택 금액을 받아 이벤트 뱃지를 적용
- `getBenefitsList()`: 받은 모든 혜택을 문자열로 가공해 반환

#### getter

- `getVisitDate()`: 방문 날짜를 반환
- `getIsAppliedEvents()`: 이벤트가 하나라도 적용되었는지를 판별해 반환
- `getBenefits()`: 적용된 혜택들의 객체를 반환
- `getIsGift()`: 증정 이벤트의 대상 유무를 반환
- `getEventBadge()`: 이벤트 뱃지 반환

### InputView

입력을 담당하는 파일

- `readDate()`: 방문 날짜 입력받아 유효성 검사 후 통과할 때 까지 반복 입력

- `readOrder()`: 주문 메뉴 입력받아 유효성 검사 후 통과할 때 까지 반복 입력

### OutputView

출력을 담당하는 파일

- `printResultDetails()`: 모든 결과를 출력
- `printGreeting()`: 인사말을 출력
- `printPreviewTitle()`: 미리보기 제목을 출력
- `printMenu()`: 주문 메뉴 목록 출력
- `printBeforeDiscountTotalAmount()`: 할인 전 총 주문 금액 출력
- `printGift()`: 증정 메뉴 유무 출력
- `printBenefitsList()`: 혜택 내역 출력
- `printTotalDiscountAmount()`: 총 혜택 금액 출력
- `printFinalPaymentAmount()`: 할인 후 예상 결제 금액 출력
- `printEventBadge()`: 이벤트 배지 유무 출력

## 예외 처리 사항

### validateInput.js

#### 방문 날짜

- [x] 숫자가 아닐 경우
- [x] 1~31 범위의 숫자가 아닐 경우

#### 메뉴 주문

- [x] 정해진 메뉴 입력 형식이 아닐 경우
  > 메뉴1-개수,메뉴2-개수,메뉴3-개수,...
- [x] 최대 주문 가능한 메뉴 개수 제한
- [x] 메뉴판에 없는 메뉴를 주문했을 경우
- [x] 메뉴의 개수가 숫자가 아니거나 1 이상이 아닐 경우
- [x] 이미 주문한 메뉴를 중복해서 입력했을 경우
- [x] 음료만 주문했을 경우

# 4주차 미션 요구사항 정리

- 날짜별로 해당되는 이벤트 항목 정의
- 이벤트 항목들 객체에 저장 후 번호로 출력

## 1. 미션 내용

- 이메일 형식의 기능 요구 사항.
- 구현에 필요한 요구사항, 배경지식은 이메일 내용에 전부 담겨있으니 꼼꼼히 확인.
- 필요하다면 문제의 내용을 통해 유추하고 스스로 판단해 구현하면 됨.
- 문제의 모든 내용은 충분한 검토를 거쳤고, 출제 의도를 담은 내용이다.

## 2. 이메일 내용 요약

- 발신자: 비즈니스팀
- 수신자: 개발팀

- 12월 1년 중 가장 큰 이벤트 개최 예정
- 예산 걱정 X

- 메뉴는 4개 카테고리에 총 12종의 메뉴가 존재.

  1. 카테고리: <> 로 감싸져있고, 각 카테고리는 공백 라인으로 구분

     > ex) <애피타이저>

  2. 메뉴: 콤마(, )로 구분되어 있고, 메뉴명 띄어쓰기 X, 금액은 3자리마다 콤마 존재.
     > ex) 양송이수프(6,000), 타파스(5,500), 시저샐러드(8,000)

## 3. 이벤트 정보

- 이벤트 목표

  1. 중복 할인, 증정을 허용해 고객들이 많은 혜택을 받는다고 체감하는 것
  2. 12월이 지난 5년 중 최고의 판매 금액을 달성
  3. 이벤트 참여 고객의 5%가 내년 1월 새해 이벤트에 재참여하는 것

- [x] 1. 크리스마스 디데이 할인

  - 기간: 23년 12월 1일 ~ 25일
  - 1,000원(1일)으로 시작해 날마다 100원씩 할인 금액 증가
  - 총 주문 금액에서 해당 금액만큼 할인
    > ex) 1일 1,000원, 2일 1,100원, ..., 25일 3,400원

### 다음은 12월 31일까지 진행

- [x] 3. 주말 할인 (금, 토): 메인 메뉴 1개당 2,023원 할인
- [x] 4. 특별 할인: 달력에 별이 표기된 날짜에 총 주문 금액에서 1,000원 할인
     > 일요일 + 크리스마스 당일 (3, 10, 17, 24, 25, 31)

## 4. "12월 이벤트 플래너" 개발 요청 사항

- [x] 고객들이 방문 날짜와 메뉴를 미리 선택했을 때 다음 7개의 내용 보여줄 것.

  - [x] 1. 주문 메뉴
  - [x] 2. 할인 전 총 주문 금액
  - [x] 3. 증정 메뉴
  - [x] 4. 혜택 내역
  - [x] 5. 총 혜택 금액
  - [x] 6. 할인 후 예상 결제 금액
  - [x] 7. 12월 이벤트 배지

- [x] 사용자 입력

  - [x] 모든 에러 메시지는 [ERROR] 로 시작

  1. 12월 중 식당 예상 방문 날짜는 언제인가요?

     - [x] 1 이상 31 이하의 숫자가 아니라면 에러 발생  
            `'[ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요.'`

  2. 주문하실 메뉴와 개수를 알려주세요.

     - 에러 메시지  
       `'[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.'`

       - [x] 메뉴판에 없는 메뉴 입력했을 때
       - [x] 1 이상의 숫자가 아닐 때
       - [x] 메뉴 형식이 예시와 다를 때
       - [x] 중복 메뉴를 입력했을 때
         > ex) 시저샐러드-1, 시저샐러드-1

- 출력

  - [x] 1. 주문 메뉴의 출력 순서는 자유롭게
  - [x] 2. 총 혜택 금액에 따라 이벤트 배지의 이름 다르게 출력
  - [x] 3. 총 혜택 금액 = 할인 금액의 합계 + 증정 메뉴의 가격
  - [x] 4. 할인 후 예상 결제 금액 = 할인 전 총 주문 금액 - 할인 금액
  - [x] 5. 증정 메뉴 이벤트에 해당하지 않으면 `'없음'`
  - [x] 6. 혜택 내역

    - [x] 적용된 이벤트 내역만 출력
    - [x] 하나도 없다면 `'없음'`
    - [x] 여러 이벤트 적용 시 출력 순서는 자유롭게

  - [x] 7. 이벤트 배지가 부여되지 않는 경우 `'없음'`
